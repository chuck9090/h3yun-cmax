# 待完善功能

- [ ] 优化构建项目 Webview 表单的主题适配。
   当前应用编码、`enginecode`、`h3_token` 等输入框使用浅色背景，在 VSCode 深色主题下观感不一致。后续可改为使用 VSCode Webview Theme CSS Variables，例如 `--vscode-input-background`、`--vscode-input-foreground`、`--vscode-input-border`，以自动适配用户当前主题。

- [ ] 优化字段表
   当前的字段表就只有字段编码和名称，没有控件类型和值类型，关联表单控件也没指明对应的表单编码，需要优化。

- [ ] 增加“查询对应表单名称”编辑器右键菜单命令。
   当用户在编辑器中选中一段文本时，在 Editor Context Menu 中显示“查询对应表单名称”。命令执行后先在当前应用目录的 `cmax.json` 中匹配该文本是否为表单编码；如果匹配成功，则直接打开对应表单目录下的 `fields.md`。如果本地配置未匹配到，则调用氚云平台接口查询该编码对应的表单信息，并向用户展示查询结果。

- [ ] 为 `.sql` 文件增加“从氚云查询”编辑器标题栏按钮。
   当当前活动编辑器是 SQL 文件时，在 Editor Title Menu 中显示“从氚云查询”按钮。点击后读取用户当前选中的 SQL 文本，调用氚云平台接口执行查询，并通过 Webview、Output Channel 或虚拟文档展示查询结果。

- [ ] 为 `.cs` 文件中的 SQL 片段增加“从氚云查询”右键菜单命令。
   当用户在 C# 文件中选中一段文本，且选中文本去除前导空白后以 `select` 开头时，在 Editor Context Menu 中显示“从氚云查询”。点击后复用 SQL 查询能力，调用氚云平台接口执行选中的 SQL，并在 VSCode 中展示结果。

- [ ] 探索“编译代码到氚云”表单目录右键菜单命令。
   在 Explorer Context Menu 中为表单目录增加“编译代码到氚云”命令。点击后读取本地表单前端、表单后端、列表前端、列表后端代码，并尝试调用氚云接口保存到云端，以减少用户手动复制粘贴。该能力依赖氚云相关保存接口的可用性和稳定性，前期仅作为可行性探索，实现风险较高。
