# H3Yun CMax - 氚云代码同步工具

一个 VSCode 插件,用于自动从氚云低代码平台同步应用代码到本地,实现版本管理、代码智能提示和 AI 辅助开发。

## 功能特性

- **自动构建项目**: 输入应用编码和 Token,自动下载应用下所有表单的字段表、表单代码和列表代码
- **智能文件夹管理**: 自动处理同名文件夹冲突,追加序号后缀
- **增量同步**: 支持右键点击应用文件夹进行增量同步,并可在同步完成后自动提交 Git 变更
- **Token 管理**: Token 单独保存到 `.h3token`,并自动通过 `.gitignore` 忽略;Token 缺失或失效时会提示重新输入
- **配置持久化**: 自动生成 `cmax.json` 配置文件,记录应用和表单结构信息
- **失败节点报告**: 节点表单结构请求失败时,自动在应用根目录生成 `failed-nodes.md`
- **进度显示**: 实时显示同步进度和状态

## 安装

1. 在 VSCode 中,点击左侧活动栏的扩展图标(或按 `Ctrl+Shift+X`)
2. 点击扩展面板右上角的 **"..."** 菜单
3. 选择 **"从 VSIX 安装..."**
4. 选择 `h3yun-cmax-0.0.1.vsix` 文件
5. 等待安装完成
6. **重启 VSCode**(重要!)

## 首次使用:获取 h3_token

在使用插件前,您需要先从浏览器中获取氚云的认证 Token:

### 步骤一:登录氚云

1. 打开浏览器(推荐使用 Chrome 或 Edge)
2. 访问氚云平台并登录您的账号

### 步骤二:打开开发者工具

1. 按 `F12` 键打开开发者工具
2. 或者右键点击页面任意位置,选择"检查"

### 步骤三:查找 h3_token

**Chrome/Edge 浏览器:**
1. 切换到 **Application**(应用程序) 标签
2. 在左侧菜单展开 **Storage**(存储) → **Cookies**
3. 点击氚云平台的域名
4. 在右侧列表中找到 **h3_token**
5. 复制其 **Value**(值) 列的内容

**Firefox 浏览器:**
1. 切换到 **Storage**(存储) 标签
2. 展开 **Cookies**
3. 点击氚云平台的域名
4. 找到 **h3_token** 并复制其值

### 步骤四:保存 Token

将复制的 Token 妥善保存,首次构建项目时需要输入。

**注意**: Token 有一定有效期,如果后续同步时提示 Token 失效,请重复上述步骤获取新的 Token。

## 使用方法

### 首次构建项目(命令面板)

1. 按 `Ctrl+Shift+P` 打开命令面板
2. 输入 "从氚云构建项目" 或 "氚云"
3. 选择对应命令
4. **在弹出的表单窗口中**:
   - 输入氚云应用编码
   - 点击"📖 如何获取 h3_token?"查看详细指南
   - 输入从浏览器获取的 h3_token
   - 点击"开始构建"
5. 插件会自动验证 Token 有效性
6. 验证通过后开始同步,等待完成

### 增量同步(右键菜单)

1. 在资源管理器中右键点击已同步的应用文件夹
2. 选择 **"从氚云同步"**
3. 等待同步完成(Token 已自动保存到 `.h3token`,无需重复输入)

**注意**: 
- 初次构建项目**只能使用命令面板**方式
- 增量同步可以使用右键菜单或命令面板

## 智能表单验证

插件内置了智能验证机制,确保您输入的信息正确无误:

### 实时验证

当您点击"开始构建"后,插件会:
1. **验证应用编码**: 检查应用是否存在
2. **验证 Token**: 尝试连接氚云平台验证 Token 有效性
3. **即时反馈**: 如果有错误,会在对应字段下方显示具体错误信息

### 错误提示

- **应用编码错误**: "应用编码不存在,请检查后重试"
- **Token 失效**: "Token 已失效或无效,请重新获取"
- **网络错误**: 显示具体的网络连接问题

### Token 获取指南

在表单中点击 **"📖 如何获取 h3_token?"** 链接,会打开详细的图文指南,包括:
- 如何在不同浏览器中查找 Cookie
- Token 的位置和复制方法
- 常见问题解答

## Token 失效处理

如果同步时 Token 已失效,插件会:
1. 检测到认证失败(401 错误)
2. 弹出提示告知 Token 已失效
3. 提供"重新输入 Token"按钮
4. 输入新 Token 后自动继续同步
5. 新 Token 会自动保存到 `.h3token`

## 智能同步策略

插件采用智能的同步策略,确保您的代码安全:

### 字段表 (fields.md)

**策略**: 直接覆盖

- 字段表只在氚云平台中修改
- 同步时会直接用远程版本覆盖本地文件
- 无需用户确认

### 代码文件 (form-frontend.ts, form-backend.cs 等)

**策略**: 差异检测 + 用户选择

同步时会:
1. **自动检测差异**: 对比本地和远程文件内容
2. **无差异**: 跳过,不做任何修改
3. **有差异**: 弹出对话框让您选择

### 冲突处理方式

当检测到代码文件有差异时,您可以选择:

#### 方式一: 批量处理(推荐)

如果有多个文件存在差异,插件会先统计数量,然后让您选择:

- **全部使用本地版本**: 保留所有本地修改,不覆盖任何文件
- **全部使用远程版本**: 用氚云平台的代码覆盖所有本地文件
- **逐个询问**: 每个文件都单独弹窗让您选择

#### 方式二: 逐个处理

对每个有差异的文件,插件会显示详细信息:
- 文件名
- 本地版本: XX 行 (XX 字节)
- 远程版本: XX 行 (XX 字节)
- 变化: +X 行, -X 行

然后您可以选择:
- **$(file-code) 使用本地版本**: 保留我的修改
- **$(cloud-download) 使用远程版本**: 覆盖为氚云版本
- **$(skip) 跳过**: 暂不处理,下次同步时再决定

### 同步流程

```
开始同步
  ↓
获取最新表单列表
  ↓
预先检测所有文件差异
  ↓
发现冲突? → 是 → 询问用户如何处理(批量/逐个)
  ↓           ↓
  否        根据用户选择处理
  ↓           ↓
直接同步    应用用户选择
  ↓           ↓
更新配置文件
  ↓
同步完成
```

### 最佳实践

1. **在氚云平台修改字段**: 同步时会自动更新 `fields.md`
2. **在本地修改代码**: 同步时会提示您选择保留哪个版本
3. **定期同步**: 建议每次在氚云平台修改后及时同步
4. **使用版本控制**: 建议将项目纳入 Git 管理,便于追踪变更

## 同步后的项目结构

```
应用名称/
├── cmax.json                    # 项目结构配置文件(不包含 Token)
├── .h3token                     # 本地 Token 文件(自动创建,不提交 Git)
├── .gitignore                   # 自动忽略 .h3token
├── failed-nodes.md              # 节点获取失败报告(仅有失败时生成)
├── 表单1名称/
│   ├── fields.md                # 表单字段表
│   ├── form-frontend.ts         # 表单前端代码
│   ├── form-backend.cs          # 表单后端代码
│   ├── list-frontend.ts         # 列表前端代码
│   └── list-backend.cs          # 列表后端代码
├── 表单2名称/
│   └── ... (同上)
└── ...
```

## 配置文件说明

### cmax.json

插件会自动生成此文件,记录应用和表单结构信息,用于后续增量同步:

```json
{
  "appCode": "应用编码",
  "appName": "应用名称",
  "forms": {
    "fxxxxx": {
      "formCode": "表单编码",
      "formName": "表单名称"
    }
  },
  "lastSyncTime": "最后同步时间(ISO格式)"
}
```

**安全提示**: 
- Token 会以明文形式保存在 `.h3token` 中
- 插件会自动在应用目录 `.gitignore` 中加入 `.h3token`
- `cmax.json` 不包含 Token,可提交到代码仓库用于记录项目结构

## 常见问题

### Q: 如何访问插件功能?

A: 
- **初次构建项目**: 只能使用命令面板
  1. 按 `Ctrl+Shift+P` 打开命令面板
  2. 输入 "从氚云构建项目" 或 "氚云"
  3. 选择对应命令

- **增量同步**: 可以使用两种方式
  1. 命令面板: 按 `Ctrl+Shift+P`,输入 "从氚云同步"
  2. 右键菜单: 在应用文件夹上右键 → "从氚云同步"

### Q: 如何获取 h3_token?

A: 
1. 在浏览器中登录氚云平台
2. 按 F12 打开开发者工具
3. 切换到 Application/存储 标签
4. 找到 Cookies 中的 h3_token
5. 复制其值

详细步骤请参考上文"首次使用:获取 h3_token"章节。

### Q: Token 会过期吗?

A: 是的,Token 有一定的有效期。当 Token 失效时:
- 插件会检测到认证失败
- 弹出提示让您重新输入 Token
- 输入新 Token 后会自动继续同步
- 新 Token 会自动保存到 `.h3token`,下次同步无需再次输入
- 如果团队成员从 GitHub clone 项目后缺少 `.h3token`,首次同步时插件会提示输入 Token 并重新生成 `.h3token`

### Q: 安装后看不到菜单项?

A: 
- **初次构建项目**: 请使用命令面板(`Ctrl+Shift+P`),输入 "从氚云构建项目"
- **增量同步**: 在应用文件夹上右键可以看到 "从氚云同步" 菜单项

请确保:
- 已重启 VSCode
- 已打开一个工作区文件夹(必须打开文件夹才能使用)

### Q: 点击命令后没有反应?

A: 可能的原因:
- 网络连接问题,无法访问氚云平台 API
- 应用编码输入错误
- Token 无效或已过期
- 氚云接口响应结构发生变化,需要调整 `src/parsers` 下对应解析器

### Q: 如何处理同名文件夹?

A: 插件会自动处理同名冲突:
- 如果已存在 "应用名称" 文件夹,会创建 "应用名称-1"
- 如果 "应用名称-1" 也存在,会创建 "应用名称-2",以此类推

### Q: Token 保存安全吗?

A: 
- Token 以明文形式保存在 `.h3token` 文件中
- `.h3token` 会被插件自动加入 `.gitignore`,不要提交到公开仓库
- `cmax.json` 不再保存 Token,可以提交用于团队共享项目结构
- 团队成员 clone 项目后,同步时输入自己的 Token 即可生成本地 `.h3token`

### Q: 如何卸载插件?

A: 
1. 打开扩展面板(`Ctrl+Shift+X`)
2. 搜索 "H3Yun CMax"
3. 点击插件右侧的齿轮图标
4. 选择 "卸载"
5. 重启 VSCode
